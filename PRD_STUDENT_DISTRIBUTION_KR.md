# ResearchOS 학생 배포 PRD (v1)

## 1) 제품 한 줄 정의
Zotero에 저장한 논문을 Obsidian 연구 카드/인덱스로 자동 변환해, 대학생이 문헌정리 시간을 줄이고 글쓰기 시작 속도를 높이는 개인 연구 OS.

## 2) 문제 정의
- 논문은 모이는데 요약/정리가 밀려서 읽기 부채가 쌓인다.
- Zotero, 폴더, 노트 앱이 분리되어 흐름이 끊긴다.
- 초보 사용자는 설치/경로/스크립트 실행에서 초기 이탈이 크다.

## 3) 대상 사용자
- 주 사용자: 한국어권 대학생/대학원생 (리뷰 과제, 세미나, 학위논문 준비)
- 1차 도메인: 심리학/사회과학 (타 전공 확장 가능)

## 4) 목표 (Goals)
- 10분 내 첫 카드 1개 생성
- Zotero 논문 추가 후 5분 내 Obsidian 카드 반영
- 사용자가 주 3회 이상 루틴 실행

## 5) 비목표 (Non-goals)
- 완전 자동 고품질 문단 생성 보장
- 저작권 이슈가 있는 PDF 원문 재배포
- 다중 사용자 협업 플랫폼화 (초기 버전 제외)

## 6) 핵심 사용자 시나리오
1. 사용자가 Zotero에 논문 저장
2. Better BibTeX가 `library.json` 자동 갱신
3. 사용자가 동기화 스크립트 실행
4. Obsidian에 카드/인덱스가 생성/갱신
5. 사용자가 카드 기반으로 문단 초안 작성

## 7) 기능 요구사항
### 필수 (MVP)
- Zotero export JSON 읽기
- 논문별 마크다운 카드 생성
- 인덱스 파일 생성/업데이트
- health check 스크립트 제공
- 초보자 설치 가이드 제공

### 선택 (v1+)
- AI 분석 모드 (`--ai`)
- thesis-coach 연동
- Scopus CSV 스크리닝

## 8) 성공 지표 (KPI)
- Activation: 설치 후 24시간 내 `sync_and_analyze.py --write` 1회 실행률
- Output: 첫 주 내 생성 카드 수(중앙값)
- Retention: 2주차에도 실행한 사용자 비율
- Reliability: 동기화 실패율 (에러 실행 / 전체 실행)

## 9) 배포 범위
- GitHub 공개 저장소
- 한국어 문서 중심 제공
- 로컬 실행 기반 (macOS/Linux 우선)

## 10) 리스크 및 대응
- 경로 하드코딩 리스크: `~/ResearchOS` 표준 경로 강제 + 심볼릭 링크 가이드 제공
- Python 패키지 설치 실패: health check에서 사전 감지
- AI 품질 과신: 문단 초안 검증 책임 명시

## 11) 출시 기준 (Definition of Done)
- 설치 가이드만 보고 신규 사용자 1명이 30분 내 첫 카드 생성 성공
- README에서 실행 명령 3개 이내로 핵심 루틴 이해 가능
- 기본 루틴 실행 후 Obsidian 산출물 위치가 문서와 일치

## 12) 버전 계획
- v0 (현재): 단일 사용자 로컬 자동화
- v1: 경로/설치 안정화 + 오류 메시지 정리
- v2: 설정 파일 기반 커스터마이징(경로/폴더명)
